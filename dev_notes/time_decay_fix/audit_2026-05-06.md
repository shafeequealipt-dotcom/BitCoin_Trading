## One Documented Design Choice (Not a Defect)

The plan mentioned `TIME_DECAY_STRUCT_NO_DATA` as a separate INFO event for cache-miss / cold-start. The implementation folds the no-data information into the `reason='no_data:...'` field of the existing `TIME_DECAY_STRUCT_GUARD` event. Functionally equivalent; operators filter via `grep TIME_DECAY_STRUCT_GUARD | grep "reason='no_data"`. Documented in `project_time_decay_fix_status.md` memory.
